Problem Statement

Without a review interface, AI extraction errors would silently reach school deans. Officers need to inspect what was attributed to each school/service before any action is taken. This is the MTP validation gate — the screen that lets us confirm the AI is working correctly before building MUP email dispatch.
💡

Implementation Drifts

Drift 1 — StaticPool added to conftest
SQLite in-memory DB was invisible across threads when TestClient dispatched routes. Fixed by adding poolclass=StaticPool so all connections share one in-memory DB. No impact on production code.
Drift 2 — httpx pinned to 0.23.3
starlette 0.27 TestClient incompatible with httpx 0.28. Pinned httpx==0.23.3 for test compatibility. requirements.txt updated.
Drift 3 — Dual POST /batches routes
Added a second POST /batches handler for HTML form submissions (returns redirect) alongside the JSON API handler. Both required because the trigger form uses HTML form POST while tests use JSON body.
Drift 4 — Mark as Reviewed unreachable on zero-mention batches
Found during PR review: the "Mark as Reviewed" button was nested inside the {% if recipients %} block, so a batch that completes extraction with zero entity mentions (a legitimate outcome, not an error) had no reachable button in the UI to close it out — permanently stuck in "extracted" status. Moved the button outside the recipients-table conditional so it renders for status=="extracted" regardless of mention count. Added regression test.
