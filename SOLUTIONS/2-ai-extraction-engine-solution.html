Problem Statement

What specific problem are we solving?
Officers spend 3–4 days per batch manually reading each feedback to identify which parts mention specific schools or services and whether the tone is positive or negative. This is the core bottleneck. Issue #1 now reliably ingests feedbacks into the database — this issue replaces the officer's reading step with a single DeepSeek API call per feedback.
💡

Proposed Solution

What changes will be made?
New files: app/services/entities.py — load known schools/services from SCHOOLS_DEPARTMENTS.json once at startup app/services/extraction.py — async extraction service tests/test_extraction.py — 5 unit tests with mocked OpenAI client Modified files: requirements.txt — add openai app/routers/batches.py — add POST /batches/{id}/extract endpoint ━━━ PROMPT DESIGN ━━━ System prompt lists all known entity IDs and names. User message contains all Q&A pairs from one feedback formatted as: Q1: {question_1} A1: {answer_1} Q2: {question_2} A2: {answer_2} ... Response format (json_object mode): {"mentions": [{"qa_index": 1, "entity_id": "IT", "entity_type": "school", "sentiment": "positive"}, ...]} qa_index maps back to body_json[qa_index-1] to retrieve source_question and source_answer. Empty mentions array [] means no entities found in this feedback. ━━━ CONCURRENCY ━━━ All feedbacks in a batch dispatched concurrently via asyncio.gather(). Retry once on JSON parse failure. Log and skip on second failure. A per-feedback semaphore (asyncio.Semaphore(10)) caps concurrent API calls to avoid rate-limit errors on large batches. ━━━ ENDPOINT ━━━ POST /batches/{id}/extract → queries all feedbacks for the batch → runs async extraction → creates Extraction rows for every mention found → updates batch status to "extracted" Returns: {batch_id, extractions_count, failed_count}

Implementation Drifts

Drift 1 — httpx version pinned to 0.23.3
starlette 0.27.0 (installed) is incompatible with httpx 0.28.x. Pinned httpx==0.23.3 in conftest and added StaticPool to conftest for thread-safe SQLite. No functional impact on extraction logic.
Drift 2 — _system_prompt() called per-call not pre-built
Prompt built on each _call_once() invocation rather than once at module level, to avoid stale state if entities change. Cost is negligible (string formatting).
